Programming Languages:
Applications and Design

This module has two halves.

  1. Applications: we solve three problems in three languages, and we discuss how different solutions to the same problem compare.
  2. Design: we cover basics of programming language theory, building on the languages covered in the first half

The assessment has four components.

  1. Classes [15 points]:
    1. [9 points] you will solve three problems, each in three languages
    2. [6 points] you will solve on-paper problems related to typing and semantics
  2. Programming Homework [30 poins]: you will solve a problem using any programming language you want; choose wisely
  3. Paper Presentation [25 points]: you will prepare a presentation (in groups) summarising a research paper related to programming languages
  4. Two-hour examination [30 points]:
    1. [15 points] for solving (on paper) a problem in two languages, and comparing the solutions
    2. [15 points] for programming language theory

Note that the points add up to 100. The module spec should have the numbers as above. SDS should have the coursework components expressed as percentages of total coursework; for example, the dependency solver programming homework should be listed as 30/(15+30+25)≈43%.
